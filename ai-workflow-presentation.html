Anatomy of an Agent


# Persistence Developer Agent

## Purpose
Creates JPA entities, repositories, Liquibase changelogs...

## Scope
**IN SCOPE:**
- JPA entities with proper annotations
- Spring Data repository interfaces
- Liquibase XML changelogs
- Repository tests

**OUT OF SCOPE:**
- REST controllers (use REST API agent)
- Business logic (use Domain Logic agent)

## Code Templates

### Entity Template
```java
@Entity
@Table(name = "{table_name}")
@Data
@EntityListeners(AuditingEntityListener.class)
public class {Entity} implements Persistable<{Entity}Id> {
    @EmbeddedId
    private {Entity}Id id;
    // ...
}
```

This is YOUR team's knowledge, documented and reusable

Why Specialized Agents?

Benefit Explanation
Focus Each agent knows one thing deeply
Quality Templates enforce best practices
Consistency Same patterns across all features
Maintainability Update agent = update all future code

    .decorateSupplier(retry.decorateSupplier(supplier));

What Stays the Same?

  • ✅ Developers make all decisions
  • ✅ Code review still required
  • ✅ Tests must pass
  • ✅ Architecture ownership with team
  • ✅ You can always write code manually
AI assists, doesn't replace

When NOT to Use Agents

  • 🔧 Simple one-line fixes
  • 🔍 Highly exploratory work
  • 📚 When you need to learn the pattern
  • 🎨 When templates don't fit your case
  • ⚡ Quick prototypes

Use your judgment - agents are tools, not mandates

The Review Mindset

Treat AI output like code from a junior developer:

  • ✅ Follows patterns correctly
  • ⚠️ Might miss edge cases
  • ⚠️ Might over-engineer
  • ❓ Business logic needs validation

Always review. Always test. Always think.

Customization & Governance

These Are YOUR Agents

  • Team owns the agent definitions
  • Update templates when patterns evolve

d4e5f6g Add scheduler-agent for background jobs

How to Evolve Agents

  1. Notice repeated pattern in code reviews
  2. Update agent template
  3. All future code follows new pattern

Example:

"We decided to use Optional instead of null returns"


// Update in integration-client-developer.md:
public Optional getById(String id) {
    // ... returns Optional.empty() on 404
}

→ All future clients follow this pattern

Common Questions

"Will this replace developers?"

No. Developers design, decide, review. AI handles boilerplate.

"What about code quality?"

Same review process. AI follows your patterns.

"What if AI generates wrong code?"

You review everything. Tests must pass. Treat it like junior dev output.

More Questions

"How do we maintain agents?"

Same as documentation. Update when patterns change.

"Can we use this for other projects?"

Yes! Agents are portable. Adjust for project specifics.

Agent templates are generic patterns. Your business logic stays yours.
